# hardware/ — рабочая зона инженера (И)

Это твоя папка. Здесь живёт всё по железу: конструкция, прошивки, докшиты, протоколы экспериментов и любые рабочие заметки. Правила — минимальные, но железные (см. `AGENTS.md` в корне):
- всё значимое фиксируется в `CHANGELOG.md` со статусом (`готово` — только после теста);
- конструкция версионируется (v0, v1…), устные договорённости конструкцией не являются;
- **папка `integration/` — священна**: из неё Ф забирает данные для стыковки железа с системой.

## Структура

| Папка | Что кладём |
|---|---|
| `cad/` | 3D-модели узлов (STEP/F3D/STL): сниматель, доводчик, клей-модуль, жёлоб, крепления |
| `drawings/` | Чертежи, схемы электрики/пневматики, эскизы |
| `firmware-esp32/` | Прошивки контроллеров механики (PlatformIO/Arduino), по узлам |
| `klipper/` | Конфиги принтеров и макросы (`COOLDOWN_EJECT`, `FIRST_LAYER_CHECKPOINT`, `WIPE`, `PARK`) — совместно с Ф |
| `datasheets/` | Докшиты приводов, драйверов, датчиков, реле, весов — всё покупное |
| `experiments/` | Протоколы Э1–Э6 (ТЗ-2): таблицы циклов, замеры усилий, фото/видео, выводы |
| `integration/` | **Точка стыковки с софтом** — см. ниже |
| `notes/` | Твоя «всякая хрень»: черновики, идеи, ссылки, расчёты — без правил |

## integration/ — контракт железо ↔ софт

Сюда кладётся всё, что нужно Ф для интеграции, в актуальном виде:
- **pinout ESP32** каждого узла (какой GPIO что делает);
- **MQTT-интерфейс узла**: топики, формат payload, команды и события (согласуется с `docs/домен-модель-и-контракты.md`, разд. 4);
- **тайминги цикла**: сколько длится проход снимателя, открытие двери, стабилизация весов, нанесение клея — софт строит на них конечный автомат;
- **схемы подключения**: питание, реле, датчики — что куда воткнуто на стенде;
- калибровочные константы (датчик тока → Н·м, HX711 → граммы).

Правило: изменил железо так, что поменялся интерфейс (пин, топик, тайминг) — **обнови файл в `integration/` в тот же день** и отметь в `CHANGELOG.md`. Софт слепо доверяет этой папке.
