## 1. Цель прототипа — что доказываем

1. **Д1. Деталь стабильно снимается со стола без человека** на материалах клиента — ≥200 циклов подряд без вмешательства.
2. **Д2. Цикл адгезия/освобождение работает без человека** — по ветке А (клей-автомат), ветке Б (текстурированный PEI) или гибридом: надёжная адгезия при печати И надёжное освобождение при остывании, без ручных операций между печатями.
3. **Д3. Цикл экономически осмыслен**: пауза «остывание + снятие + рестарт» ≤10–12 мин с активным обдувом (а не 30+ мин пассивного остывания — главные грабли аналога Quinly).
4. **Д4. Отбраковка физически работает**: вердикт CV до снятия → заслонка → деталь в правильной таре, факт сброса подтверждён датчиками, учёт сходится.

Прототип НЕ доказывает: точность CV на всех дефектах (трек датасета), экономику фермы, что-либо про Bambu.

## 2. KPI стенда (перенесены из ТЗ-1; гейт на выезд к клиенту)

| # | KPI | Целевое значение |
|---|---|---|
| С1 | MTBF цикла снятия | ≥200 циклов подряд без вмешательства (PLA), затем ≥50 циклов на каждый доп. материал |
| С2 | Catch-rate на намеренном браке (Э6) | CV ловит ≥70% намеренного брака до 30% прогресса (PLA/PETG) |
| С3 | Чекпоинт первого слоя | ловит ≥ заданной доли намеренно испорченных первых слоёв до 5% прогресса (порог фиксируется до начала испытаний Э6) |
| С4 | Время цикла | остывание+снятие+рестарт ≤10–12 мин (PLA/PETG); для ABS/ASA — измерить и зафиксировать в модели планировщика |
| С5 | Учёт | 100 сбросов с перемешанными вердиктами — расхождение журнала и тар = 0 |

## 3. Стенд

### Состав
- Принтеры: **офисные Creality K1C ×2** ✅ (root → Moonraker). Представитель ABS-кейса — ❓ (Qidi, если доступен; иначе ABS-эксперименты на K1C с учётом камеры).
- Edge: Linux-миникомп x86 (боевой образ) ✅; Mac mini допустим как временный носитель до закупки.
- Камеры USB 1080p + LED-кольца ×2, реле ×2, дымовой датчик, ИБП.
- Механика: комплект «Автономия» v0 ×2 (разд. 7) + макет жёлоба с заслонкой и двумя тарами на раме + весы (платформа 5–10 кг, HX711) ×2.
- **Клей-модуль v0** (прототип инженера): насос + распылитель на каретке + бак; питание — отдельная ветка от БП, управление через ESP32 ✅ (не с коннектора вентилятора).
- Материалы: PLA, PETG, ABS/ASA — по 2 катушки; CF-варианты — ❓ по аудиту (чем печатает ФТ); текстурированные PEI-пластины 1–2 брендов; клей (водная база, кандидаты подбираются в Э1).

### Софт стенда
Боевой стек ТЗ-1 в мини-конфигурации (edge + локальный «центр» в Docker на рабочей станции): конечный автомат, макросы, **чекпоинт первого слоя**, журнал, панель. Стенд с первого дня гоняет боевой код, не скрипты-однодневки ✅.

### Конструкторская документация ✅
CAD/STL/чертежи и прошивки ESP32 живут в репо (`hardware/`), версии конструкции — v0, v1… с записью изменений в `CHANGELOG.md` по статусам `AGENTS.md`. Устные договорённости конструкцией не являются.

## 4. Программа экспериментов

### Э1. Адгезия/освобождение: клей-автомат vs текстурированный PEI (недели 1–4) 🔬
Сравнение двух веток на одинаковой эталонной детали. **Критерии выбора зафиксированы до старта:**

| Критерий | Ветка А: клей-автомат | Ветка Б: голый текстур. PEI |
|---|---|---|
| Циклов без вмешательства | ≥100 подряд | ≥100 подряд |
| Стабильность усилия снятия | дрейф в допуске привода | дрейф в допуске привода (замер каждые 25 циклов) |
| Время полного цикла | вкл. нанесение+сушку клея | без доп. операций |
| Состояние линзы камеры | после 100 циклов распыла — деградации картинки нет | n/a |
| Надёжность тракта | сопло/насос без засыхания (автопромывка), MTBF ≥200 | деградация пластины: лимит циклов/моек измерен |
| Обслуживание на 100 циклов | мойки, доливы — хронометраж | мойки IPA — хронометраж |
| Универсальность | PLA/PETG/ABS (+CF если в скоупе) | по материалам, где работает |

- Ветка А дополнительно: подбор химии клея (водная база — пожаробезопасность в камере ✅), температура стола при нанесении, дозировка по площади первого слоя, цикл автопромывки.
- Ветка Б дополнительно: 1–2 бренда пластин (сжато против v1: не 3 бренда × полная матрица), протокол моек, критерий деградации: усилие «на горячую» упало ниже порога отлипа ИЛИ «на холодную» выросло выше предела привода.
- **Допустимый исход — гибрид** ✅: PLA/PETG на PEI, остальное с клеем. Выход: выбранная схема (или гибрид), профили первого слоя на связку материал×поверхность, пороговые значения для счётчиков расходников в системе.

### Э2. Сниматель (недели 1–6) — критический путь
- Конструкция v1: две схемы — (а) рычаг на отдельном приводе (NEMA17/серва + датчик тока), (б) сметание штатной головой по макросу с насадкой-«плугом». Собрать обе.
- **Критерии выбора между схемами (фиксируются до сравнения)** ✅: повторяемость снятия на эталонной детали (≥50 циклов каждая), время прохода ≤20 с, отсутствие контакта с покрытием, совместимость с клей-модулем на каретке (для схемы б), сложность монтажа техником по инструкции, число нестандартных деталей.
- Требования: контроль усилия (порог → стоп + алерт «деталь не снялась»); концевики; никаких острых кромок к столу.
- Тепловой цикл: обдув стола 120 мм — остывание с 60–70 °C до температуры отлипа ≤6–8 мин (PLA/PETG); ABS/ASA — измерить и зафиксировать (Д3/С4). С клеем ветки А — проверить агрессивный обдув без коробления (аргумент ветки А).
- **Испытание Д1 с промежуточными точками** ✅: контроль на 50 → 100 → 200 циклах PLA (ранний сигнал вместо «узнаем на неделе 6»), затем по 50 циклов PETG/ABS/ASA. Вмешательство = сброс счётчика. Лог каждого цикла: усилие, время остывания, успех сброса.

### Э3. Сортировка и подтверждение (недели 4–8)
- Жёлоб с заслонкой-дивертером (1 серва), две тары, ИК-барьер, весы под тарами.
- Логика: вердикт до снятия → заслонка → сброс → ИК-событие в окне ±10 с → приращение веса ≈ масса детали ±20% → журнал. Любое рассогласование → стоп принтера + алерт с фото.
- Испытание Д4/С5: 100 сбросов с перемешанными вердиктами, расхождение 0; детали от 5 г до 200+ г.

### Э4. Доводчик двери (недели 6–10) — стандартный узел, не ABS-опция ✅
Парк K1 весь закрытый: дверь открывается на каждый цикл снятия при любом материале.
- Конструкция: серва/линейный привод + шарнирная тяга; открытие на снятие, закрытие перед стартом; концевик положения.
- Для ABS/ASA: термопара камеры — подтвердить, что цикл открытия не роняет температуру ниже допуска к следующему первому слою.
- Политика CV на ABS/ASA: прогон намеренного брака — только алерты, ноль авто-пауз.

### Э5. Пожарный контур стойки (недели 8–10)
Дым-имитация → локальное обесточивание ≤10 с (аппаратно, без edge и центра), алерт, видеофиксация. Порошковый модуль — монтаж и паспортная проверка (без срабатывания). Кнопка ежемесячного теста цепи — в конструкции стойки ✅.

### Э6. Датасет намеренного брака (фоново, недели 2–12)
- Протокол «убийств» печати: отклейка, спагетти (сдвиг Z/оторванная деталь), недоэкструзия, узел на катушке, blob; **отдельно — порча первого слоя** (грязь, сбитый Z-offset) для контура 5.3.1 ✅.
- Каждый прогон размечается: класс дефекта, слой начала, кадры до/после.
- Цель к пилоту: ≥300 размеченных инцидентов + ≥1000 нормальных финалов + эталоны первых слоёв деталей-эталонов. Вход для дообучения YOLO, PatchCore-эталонов и промптов VLM.

## 5. Финальные испытания стенда (недели 11–12)
**72-часовой non-stop прогон**: оба принтера, реальные детали ФТ (STL — с аудита), смешанные материалы, полный цикл с сортировкой и чекпоинтом первого слоя, имитация 2 обрывов связи и 1 дымового события. Критерии = Д1–Д4 + KPI С1–С5 + журнал сходится 1:1 с тарами. Протокол подписывается — это гейт на тиражные закупки и выезд. Статус в `CHANGELOG.md`: «готово ✓ (ручной тест: команда, дата)».

## 6. Пилотная стойка автономии у ФлайТех (после гейта стенда)
- 4–8 принтеров K1-семейства (одна модель, выбор на аудите), перевод на выбранную схему адгезии, монтаж комплектов «Автономия» + стойка (жёлоб, сортировка, пожарный модуль), edge-узел.
- Неделя супервизии (ночные смены с дежурным-наблюдателем) → 5 ночей KPI-прогона без человека (ТЗ-1, разд. 2).
- Обучение оператора ФТ: регламенты (расходники адгезии, катушки, тары), панель, бот.
- Инструкция монтажа проверяется «чужими руками»: техник, не участвовавший в разработке, собирает комплект за ≤2 ч ✅.

## 7. Состав комплектов (инженерная спецификация, без цен)

**Комплект «Контроль» (на любой принтер):** камера 1080p + крепление, LED-кольцо, реле питания (локальное управление).

**Комплект «Автономия» (на принтер K1-класса, поверх «Контроля»):**
- привод снимателя (NEMA17/серва + драйвер с датчиком тока) + рычаг/рама/направляющая (алюм. профиль + печатные детали);
- контроллер ESP32 + обвязка (MQTT to edge) ✅ (вариант «edge-GPIO» из v1 удалён — у миникомпа нет GPIO);
- вентиляторы обдува стола ×2 + крепёж;
- доводчик двери (серва/привод + тяга + концевик) — стандартно ✅;
- поверхность: PEI-пластины ×2 ИЛИ клей-модуль (насос, бак с датчиком уровня, распылитель на каретке, тракт с автопромывкой) — по Э1 🔬;
- доля жёлоба, кабели, крепёж.

**Стойка автономии (на 4–8 принтеров):** заслонка сортировки + серва, ИК-барьер, весы ×2 тары, пожарный модуль (дым-датчик + реле + порошковый модуль + кнопка теста), рама/жёлоба, кабельная обвязка.

**Edge-узел (на площадку):** Linux-миникомп x86 (массовый, класс N100/N305/Ryzen) + ИБП + коммутатор.

Уточнённый BOM с поставщиками — артефакт стенда (разд. 8). Закупочные цены и цены клиенту — вне этого документа ✅.
