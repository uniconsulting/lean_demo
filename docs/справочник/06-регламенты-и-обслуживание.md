# Блок 6. Технические регламенты и обслуживание

Для кого: все, кто касается принтеров. Уроки выживших ферм однозначны: главные причины брака — не «плохие модели ИИ», а грязные столы, разболтанные винты и забитые экструдеры. Обслуживание — это и есть надёжность.

---

## 6.1 Регламенты по периодичности (основа модуля «задачи оператора»)

**Ежедневно / каждую смену:**
- визуальный осмотр парка (панель + глаза): подтёки пластика на соплах, обрывки на столах, посторонние звуки;
- очистка столов от обрывков; проверка «стол пуст» перед ночной сменой (у нас — CV-preflight, но глаза не отменяются);
- уровень/состояние расходников адгезии (клей в баке / состояние пластин);
- тары: разобрать, сверить с журналом (наш сценарий «Учёт»).

**Еженедельно:**
- мойка рабочих поверхностей по типу (блок 5.5); осмотр покрытия на царапины/проплешины;
- продувка/чистка вентиляторов головы (забитый обдув = деградация нависаний и heat creep);
- протирка линз камер и проверка картинки (особенно при ветке А — клеевой туман);
- осмотр ремней (натяжение, потёртости), тест концевиков.

**По наработке (моточасы считает система, ТЗ-1 5.5):**

| Наработка | Работы |
|---|---|
| ~200–300 ч | смазка направляющих/винтов Z; подтяжка эксцентриков/проверка люфтов; cold pull сопла |
| ~500 ч | ревизия сопла (замер/замена — на CF-материалах чаще); чистка экструдера от крошки; калибровка датчиков (ток снимателя, весы HX711) |
| ~1000 ч | замена PTFE-трубок; ревизия вентиляторов (подшипники); проверка проводки головы на перелом (постоянный изгиб!); полная калибровка (mesh, input shaping, PA) |

**Ежемесячно:** тест пожарной цепи кнопкой (имитация дыма → обесточивание ≤10 с) — фиксируется в журнале со статусом «готово ✓(ручной тест)». Проверка ИБП.

## 6.2 Калибровки — когда и какие

- **Z-offset** — после каждой смены сопла/поверхности; проверка первым слоем (у нас — чекпоинт всё равно смотрит каждую печать).
- **Bed mesh** — после переезда принтера, замены стола, при систематических проблемах первого слоя в одной зоне.
- **Pressure advance** — на каждый новый материал/бренд (живёт в профиле филамента).
- **Input shaping** — после механических работ (ремни, масса головы — например, установили клей-модуль!).
- **Flow/поток** — при смене бренда филамента; на CF — чаще (диаметр сопла растёт от износа).
- Правило: результат калибровки — коммит в конфиг/профиль с записью в CHANGELOG, а не «покрутил и забыл».

## 6.3 Типовые неисправности → диагностика

| Симптом | Вероятные причины (в порядке частоты) |
|---|---|
| Первый слой не липнет местами | жирная/грязная поверхность → мойка; сбитый Z-offset; кривой mesh |
| Недоэкструзия/пропуски | влажный филамент; частичное забитие сопла (cold pull); heat creep; пережат путь нити; износ шестерён |
| Стрингинг вырос внезапно | влага (чаще всего); температура завышена; ретракт сбит |
| Layer shift | ремень (натяжение/зуб перескочил); клин на оси (обрывок пластика на рельсе); ток драйвера; слишком высокая скорость/ускорение |
| Спагетти | деталь оторвалась (адгезия) или суппорт рухнул; после blob'а |
| Blob на сопле | деталь частично отлипла и налипла на сопло; подтёк из-за износа сопла/термобарьера |
| Шум/скрип | сухие направляющие; вентилятор с убитым подшипником |
| «Принтер офлайн» в панели | сеть (Wi-Fi!); зависший klippy → перезапуск; питание |

Каждый разобранный случай — размеченный пример в датасет (класс дефекта, причина, кадры) — это тоже регламент.

## 6.4 Пожарная безопасность — почему это не паранойя

FDM-принтер — нагреватель 300–400 Вт с пластиком, работающий ночами без людей. Реальные пожары ферм — статистика, а не страшилка. Слои защиты (все обязательны, каждый ловит отказ предыдущего):

1. **Прошивка — thermal runaway protection**: если температура не реагирует на нагрев как ожидается (выпал термистор, залип MOSFET) — аварийное отключение. В Klipper включено конфигом (`verify_heater`) — проверяется при вводе принтера в реестр.
2. **Электрика**: исправная проводка, заземление, автоматы; никаких бытовых удлинителей-гирлянд; staggered heating (диспетчер) против пиковых перегрузок.
3. **Датчики дыма над каждой стойкой** → **аппаратное обесточивание** стойки реле ≤10 с — работает без edge, без центра, без интернета.
4. **Порошковый модуль самосрабатывающего пожаротушения** над стойкой — последний рубеж, автономный, сертифицированный.
5. **Система**: алерт при дыме/перегреве/потере термодатчика; видеофиксация; реле-сценарии.
6. **Организация**: огнетушители на площадке, свободные проходы, разграничение ответственности в договоре, ежемесячный тест цепи (6.1).

Запрещено: обходить thermal runaway, печатать с неисправным термистором «до конца смены», блокировать дымовой датчик («он пищит от ABS» — значит вентиляция, а не скотч на датчик).

## 6.5 Ввод нового принтера в систему (чек-лист)

1. Root/Moonraker по официальной процедуре; фиксация версии прошивки.
2. Конфиг: `verify_heater`, max-температуры, датчики филамента; наши макросы.
3. Сеть: DHCP-резервация, паспорт в реестре (модель, стол, сопло).
4. Комплект «Контроль»: камера (стол в кадре, фокус, LED), реле; проверка картинки в панели.
5. Для автономных: комплект «Автономия» по инструкции монтажа (≤2 ч техником), калибровка датчика тока, тест цикла снятия ×5 под присмотром.
6. Калибровки: mesh, Z-offset, PA на дежурном материале; тестовая деталь-эталон.
7. Запись в CHANGELOG со статусом «готово ✓(ручной тест: кто/когда, тест-печать + цикл снятия)».
