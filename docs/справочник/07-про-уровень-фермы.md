## 7.1 Экономика цикла: почему секунды имеют значение

Производительность автономного принтера определяется не скоростью печати, а **полным циклом**:

```
T_цикла = T_печати + T_остывания + T_снятия + T_рестарта(+нанесение клея, если ветка А)
```

На серийных мелких деталях (30–90 мин печати) паразитная часть цикла решает всё: пассивное остывание ~30 мин (как у аналога Quinly) на часовой печати = −33% производительности фермы. Наши цели (ТЗ-2, Д3): активный обдув стола → остывание ≤6–8 мин (PLA/PETG), полный паразитный цикл ≤10–12 мин. Для ABS/ASA остывание дольше по физике (стол 90–110 °C + нельзя шокировать деталь) — ночью это некритично, но время фиксируется в модели планировщика: диспетчер должен знать реальное T_цикла каждой связки «изделие × материал», иначе план ночи — фантазия.

Второй неочевидный фактор — **раскладка стола**: полный стол мелочи эффективнее по времени печати, но одна отвалившаяся деталь портит весь стол (классика: сбитая деталь таскается соплом по соседям). Уроки выживших ферм: мелкие партии 4–9 деталей на стол — оптимум надёжности; сериями управляет очередь, а не жадность раскладки.

## 7.2 Первый слой: инженерия, не магия

До 70% брака закладывается в первом слое. Что реально влияет (по убыванию):
1. **Состояние поверхности** — жир/пыль/остатки клея. Правило «не трогать пальцами» + мойки по регламенту.
2. **Z-offset** — сотые мм. У K1-семейства автокалибровка тензодатчиками, но она не отменяет дрейф (нагар на сопле, смена пластины). Симптомы: волны/задиры = близко; «спагетти-лапша» вместо линий = далеко.
3. **Температуры первого слоя** — обычно +5–10 °C к рабочей (лучше смачивание), обдув выключен/минимален.
4. **Скорость первого слоя** — медленно (20–35 мм/с эквивалент): расплаву нужно время «вжаться».
5. **Геометрия пятна контакта** — мелкое пятно = рычаг для отрыва; brim/якоря — приём против отрыва углов (наш умный пост-процессинг — этап 2).

**Наш чекпоинт первого слоя (ТЗ-1 5.3.1)** стоит на трёх фактах: (а) дефект первого слоя почти всегда виден сверху сразу после его завершения; (б) чекпоинт — редкое событие, можно позволить тяжёлую проверку каждой печати; (в) серийность даёт эталоны: та же деталь, тот же ракурс, тот же свет → сравнение с «золотыми» кадрами (PatchCore-класс) ловит отклонения без разметки брака. Отмена на 2–5% прогресса стоит копейки; спагетти на 60% — часы и риск blob'а.

## 7.3 Автоснятие: три архитектуры и их честные цены

(Разбор рынка — `docs/analysis/`, раздел 2.)

1. **Сметание** (наш путь + 3DQue/Infinity Flow): остывший стол → рычаг/голова спихивает деталь. Дёшево, ретрофитится. Цены: тюнинг адгезии на грани («бритва»: недодержал/передержал»), потеря площади стола под ход сметателя, **двусторонние отказы** — деталь отвалилась в печати ИЛИ не оторвалась на съёме. Наши ответы: контроль усилия (датчик тока → стоп + алерт), подтверждение сброса (ИК + весы), вердикт до снятия, клей как управляемый разделитель (ветка А).
2. **Плейт-своп** (Prusa AFS, покойный Voodoo): робот меняет пластину целиком, деталь отделяется офлайн. Адгезия без компромиссов, любые материалы — но дорого, робот на кластер, отделение всё равно чьё-то. Наш enterprise-трек, не MVP.
3. **Конвейерные принтеры** (belt): печать «в бесконечность» со сползанием с ленты. Ниша длинномеров; качество/номенклатура ограничены — не наш случай.

Отраслевой факт для смирения: Prusa держит 600+ принтеров с **ручным** съёмом. Автоснятие оправдывается только серийностью и ночами — ровно наш кейс, поэтому KPI и стенд, а не вера.

## 7.4 CV на ферме: что реально работает

- **Потолки детекции** (бенчмарки + практика Bambu/Obico): даже хорошие модели дают ~90% precision/recall в идеальных условиях. Продуктовая метрика — пара «% пойманного брака до N% прогресса» × «% ложных остановок», и **бюджет ложных остановок ≈ 0 для ABS/ASA** (ложная пауза сама делает брак). Отсюда каскад с VLM-подтверждением и полный запрет авто-пауз на ABS/ASA.
- **Свет решает больше модели**: постоянный рассеянный LED убирает главный источник ложняков (тени, блики, «ночная слепота», авто-экспозиция). Фиксированная экспозиция камеры обязательна.
- **Классика ложных срабатываний** (знать в лицо): вазы/цилиндры «похожи на спагетти», отражения экструдера, логотипы на пластинах, обрывки прошлых печатей, суппорты сложной формы, полупрозрачный пластик.
- **Датасет — актив**: намеренный брак на стенде (Э6) даёт чистую разметку дёшево; каждое «человек подтвердил/опроверг вердикт» — бесплатная разметка из эксплуатации. Катастрофы (спагетти/blob) важнее для датасета, чем косметика: система в первую очередь страхует ночь.
- **Blob of doom** — самый дорогой пропуск (может убить голову и утащить проводку): контроль «нароста у сопла» — отдельный класс, при уверенной детекции пауза оправдана даже на ABS (потеря детали < потери головы) — порог согласуется в политике реакций.

## 7.5 Klipper продвинуто: наш инструментарий автономии

- **Макросы с логикой**: Jinja2-шаблоны с параметрами (`COOLDOWN_EJECT TARGET_TEMP=32`), условиями и доступом к состоянию принтера. Правило проекта: макрос **идемпотентен** (повторный вызов не ломает состояние) и **сообщает результат** (через `RESPOND`/статусные переменные — edge должен знать, чем кончилось).
- **`delayed_gcode`** — отложенные/периодические действия на стороне принтера (страховочные таймауты: «если через N мин статус не сменился — сообщи»).
- **`save_variables`** — переменные, переживающие ребут (счётчик циклов пластины — дублируется в систему, но локальная копия спасает при потере связи).
- **`exclude_object`** — исключение объекта из печати на лету: испорчена одна деталь из раскладки — убить её, достальные допечатать. Мощный приём для полных столов; интеграция с вердиктом CV — кандидат этапа 2.
- **Виртуальные пины/GPIO хоста** — сигналы между klippy и нашим агентом без физических проводов.
- **Ограничение**: Klipper не «многозадачен» — во время печати сложные манёвры ограничены; всё тяжёлое (решения, расписания) живёт на edge, принтеру — короткие детерминированные команды.

## 7.6 Датчик тока как измеритель усилия (сниматель)

Ток шагового мотора ≈ момент на валу. Драйверы TMC отдают StallGuard-метрику нагрузки — калибруем в Н·м на рычаге (Э2: динамометр-протяжка) и получаем из привода снимателя **бесплатный датчик усилия**: норма отлипа / «прилипла сильнее обычного» (ранний сигнал деградации поверхности или дрейфа профиля — тренд в телеметрию!) / порог «стоп, не деру» (защита детали, покрытия и механики). Дрейф калибровки — в регламент (блок 6.1). То же мышление применяем везде: серва доводчика с обратной связью по току = «дверь встретила препятствие».

## 7.7 Ночная смена как операционный продукт

Безлюдная ночь — это не «нажали старт и ушли», а протокол:
1. **«Зарядить ночь»** (вечерний чек-лист системы): очередь ≥12 ч с учётом реальных T_цикла; филамента хватает на всё задание каждого принтера (стартовать печать, на которую не хватит нити, — запрещено диспетчером); расходники адгезии в норме; тары пусты; все автономные принтеры прошли preflight; пожарная цепь живая (heartbeat).
2. **Политики ночи жёстче дневных**: сомнительный preflight = принтер пропускается (не будим человека из-за одного принтера); деструктив только по подтверждению (Telegram-пуш дежурному — «подтверди отмену», не «приезжай»).
3. **Деградация без интернета** — по правилам ТЗ-1: ночь доигрывается локально.
4. **Утро** — разбор: сводка, инциденты с кадрами, сверка тар с журналом, задачи. Метрики ночи (напечатано, брак, ложные остановки, вмешательства) — это и есть наши KPI в ежедневном режиме.

## 7.8 Масштабирование фермы: уроки чужой кровью

(Полные разборы — `docs/analysis/`, раздел 4.)
- **Voodoo Manufacturing** (160+ принтеров, робот-плейтсвоп, −2020): парк дешёвых потребительских машин = лавина обслуживания; обещания «400 принтеров на человека» не пережили реальность. Мораль: надёжность цикла > красота автоматизации.
- **NVBOTS**: автоснятие как отдельный продукт не выжило — только как часть системы (очередь+контроль+учёт). Ровно наш тезис.
- **Выжившие фермы**: доведи ОДИН принтер до идеала → потом масштабируй; причины брака — обслуживание, не рокет-саенс; бенчмарк лучших ~3 часа человеческого труда на 1000 часов печати — наша путеводная метрика.
- **Экосистемный риск**: Bambu закрыл API и раздал бесплатный ферм-менеджер — «просто софт для ферм» мертворождён; ценность в физическом цикле + автономности + расходниках. Klipper-парк — фундамент гарантий, потому что открыт по определению.

## 7.9 Трассируемость как enterprise-фундамент

Для производителя БПЛА вопрос «из какой партии филамента и на каком принтере напечатан этот кронштейн» — не любопытство, а приёмка. Наша цепочка (экземпляр ← задание ← изделие ← слайс-артефакт ← катушка/партия ← принтер ← поверхность ← исход ← фото ← вес) закладывается в модель данных с первого дня, потому что дописать её потом — значит переделать всё. Из неё бесплатно вырастают: отчёты смены, паспорта деталей, генеалогия материала, MES-lite (этап 2+), и датасет с полным контекстом каждого кадра.
