# Блок 3. Железо и составляющие

Для кого: все; база для И (его повседневность) и для Ф (интеграция датчиков/приводов в систему).

## 3.1 Хотэнд и сопла

- **Сопло**: латунь (стандарт, дёшево, лучшая теплопроводность), закалённая сталь (обязательна для абразивных CF/GF-филаментов — латунь они разъедают за десятки часов, диаметр отверстия растёт → плывёт экструзия), рубин/карбид (дорогая экзотика). Диаметры: 0,4 стандарт; 0,6–0,8 — производительность и прочность (толще линии) ценой детализации. **У K1C штатно закалённое сопло** — CF-материалы можно сразу.
- **Термистор** — датчик температуры сопла. Его отказ/выпадение — классический сценарий пожара (см. thermal runaway, блок 6).
- **Heatbreak/термобарьер** — узкое место между горячей и холодной зоной. Забитие из-за «теплового расползания» (heat creep) — типовая причина недоэкструзии.

## 3.2 Экструдер и подача

- Директ-экструдер (наш случай): мотор с редуктором прямо на голове. Изнашиваемое: прижимные шестерни (грызут абразив), прижимная пружина.
- **Датчик окончания филамента** (концевик: нить есть/нет) и **датчик движения** (нить реально едет? ловит запутывание и проскальзывание) — Klipper поддерживает оба штатно, наши события «runout» приходят из них.
- Путь нити: катушка → (сушилка/бокс) → трубка PTFE → экструдер. Перегибы и трение по пути = недоэкструзия; на 5-кг катушках (этап 2) это главная проектная забота.

## 3.3 Стол и поверхности (наша ключевая тема — Э1)

Сэндвич: алюминиевая плита с силиконовым нагревателем → магнитная подложка → **съёмная пружинная сталь с покрытием**:

| Поверхность | Поведение |
|---|---|
| **Гладкий PEI** (лист/напыление) | Универсальный; PETG липнет СЛИШКОМ сильно (вырывает куски покрытия — нужен разделитель, исторически клей); зеркальный низ детали |
| **Текстурированный PEI** (порошковое напыление) | Наша ветка Б: держит горячим, отпускает холодным; низ детали шершавый; деградирует от жира/износа — учёт циклов и моек |
| **Стекло/G10-стеклотекстолит** | Ровное, дёшево; G10 — вариант для ABS; тяжелее, дольше греется |
| **PA/спец-покрытия** | Под нейлоны и экзотику (вне MVP) |

Физика адгезии/освобождения — в блоке 5.5; инженерия автоснятия — блок 7.

## 3.4 Приводы и электрика движения

- **Шаговые моторы NEMA17** — стандарт осей и нашего снимателя. Работают без обратной связи: пропуск шагов (перегруз/клин) остаётся незамеченным прошивкой → layer shift.
- **Драйверы TMC** (2209 и семейство): тихие, программируемый ток, **StallGuard** — датчик нагрузки по току, умеет и хоуминг без концевиков, и наше «контроль усилия снимателя» (превысил порог → деталь не отлипла → стоп).
- **Сервоприводы** (модельные SG90/MG996/DS-серии): позиционирование по углу — наши заслонка сортировки и доводчик двери. Дешёвые пластиковые редукторы дохнут — на циклическую работу брать металл-редуктор с запасом по моменту.
- **Ремни GT2** — растягиваются со временем (регламент натяжения), **линейные рельсы/валы** — смазка по регламенту.

## 3.5 Контроллеры и наша периферия

- **Плата принтера** — STM32/подобное с MCU-прошивкой Klipper; хост klippy — Linux-модуль внутри K1.
- **ESP32** — наш контроллер механики: Wi-Fi/Ethernet, десятки GPIO, PWM для серв, АЦП; прошивается через PlatformIO. Говорит с edge по **MQTT** (топики/payload — `hardware/integration/`). Правило проекта: механика НЕ управляется с edge-GPIO (у миникомпа их нет) — только через ESP32.
- **HX711** — АЦП для тензодатчиков: наши весы под тарами (подтверждение сброса + «тара полна» + калибровка остатка катушек). Дрейфует с температурой — тарировать по расписанию.
- **ИК-барьер** (излучатель+приёмник) в жёлобе — факт пролёта детали.
- **Реле питания**: механическое реле щёлкает и изнашивается, **SSR (твердотельное)** — бесшумно и живёт дольше, но греется (радиатор) и в пробое остаётся ВКЛЮЧЁННЫМ — для пожарного контура схема продумывается с И (нормально-разомкнутая логика: пропало управление = обесточено). Управление — локальное (ESP/MQTT), никаких китайских облаков.
- **Дымовые датчики** — над каждой стойкой, завязаны на обесточивание аппаратно (работает без edge и центра).

## 3.6 Камеры и свет

- **USB-камеры (UVC)**: дёшево, без настройки сети; ограничения — длина кабеля ~5 м, полоса и число устройств на хост. Для стенда — идеально; для фермы на 80 машин физически не масштабируется на один узел.
- **IP-камеры (RTSP)**: питание и данные по одному кабелю с **PoE**, длина до 100 м, масштабируется коммутаторами; дороже, надо декодировать потоки. Решение для фермы — ❓ по аудиту (ТЗ-1).
- Требования к постановке: стол заполняет кадр, фокус фиксированный, **LED-кольцо** — постоянная освещённость day/night. Свет — главная защита от ложных срабатываний CV (блики, тени, «ночная слепота»). Автоэкспозицию фиксировать: мигание экспозиции CV воспринимает как «что-то изменилось».
- Линза загрязняется (пыль, пары пластика, а в ветке А — клей): протирка по регламенту + контроль качества картинки в Э1.

## 3.7 Питание и инфраструктура

- Принтер FDM жрёт 150–400 Вт пиково (нагрев стола), ~100–150 Вт крейсерски. **Пик — при старте нагрева**: одновременный старт 8 принтеров = 3+ кВт скачок → наш staggered heating (ступенчатый пуск от диспетчера).
- **ИБП** — обязателен для edge-узла и коммутатора (принтеры при аварии сети пусть гаснут — печать потеряна, но ферма под контролем и журнал цел). Klipper умеет резюмить печать после сбоя питания, но на это не закладываемся.
- Проводка площадки, заземление, автоматы — раздел требований к площадке в договоре (пожарный блок ТЗ-1 5.7).

## 3.8 Правило проекта по закупкам

Только **массовое железо** из розницы с несколькими поставщиками: NEMA17, TMC-драйверы, ESP32, HX711, SG90/MG996, реле, PEI-пластины, USB/IP-камеры — всё это commodity. Никаких компонентов «только под заказ с одного склада» — иначе одна пропавшая позиция останавливает тираж (ТЗ-2 разд. 7, AGENTS.md принцип 3).
