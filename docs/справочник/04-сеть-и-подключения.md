# Блок 4. Сеть и подключения

Для кого: Ф — повседневность; И — подключение узлов; Д — понимать, откуда берутся live-камеры и статусы.

---

## 4.1 Топология площадки

```
Интернет ← WireGuard-туннель (исходящий!) ← Роутер площадки
                                              │
                                        Коммутатор(ы)
                    ┌──────────────┬───────────┼────────────┬──────────────┐
                edge-миникомп   принтеры K1  IP-камеры    ESP32-узлы    рабочее место
                (+ИБП)          (Ethernet/   (PoE, если   (Wi-Fi/Eth)   оператора
                                 Wi-Fi)       выберем)                  (браузер)
```

Правила:
- **Провод всегда лучше Wi-Fi**. 80 Wi-Fi-клиентов в промзоне (металл, помехи, соседние сети) = случайные потери связи, которые выглядят как «система глючит». Минимум: edge, коммутаторы, камеры — только провод; принтеры — провод где физически возможно.
- **DHCP-резервации** (фиксированный IP по MAC) всем участникам: адрес принтера — часть его паспорта в реестре. mDNS/«авто-обнаружение» на 80 устройствах ненадёжно.
- Изоляция: ферма живёт в своей подсети/VLAN, гостевой трафик клиента отдельно. Для режимных клиентов (БПЛА) — обязательное требование ИБ.
- **NTP (время)**: все часы синхронизированы через edge. Рассинхрон времени = каша в журналах и трассируемости.

## 4.2 Протоколы, которыми говорит система

| Протокол | Где у нас | Суть |
|---|---|---|
| **WebSocket (Moonraker)** | edge ↔ каждый Klipper-принтер | Постоянное соединение: подписка на статусы + команды/макросы |
| **MQTT** | шина площадки: edge ↔ ESP32, внутренние события | Лёгкий pub/sub через брокер (Mosquitto на edge). Темы вида `len/{site}/{unit}/...` |
| **RTSP/MJPEG** | камеры → edge | Видеопотоки; edge декодирует и прореживает кадры |
| **HTTP/REST** | панель ↔ центр; edge ↔ центр | API, загрузка файлов |
| **WireGuard** | площадка ↔ центр | Шифрованный туннель |
| **USB-serial** | не используем в MVP | Наследие Marlin |

## 4.3 MQTT — минимум для работы

- **Брокер** (у нас Mosquitto на edge) — почтальон: издатели шлют в **топики**, подписчики получают. Никто ни с кем не соединён напрямую — узлы можно перезапускать независимо.
- **QoS**: 0 — «выстрелил и забыл» (телеметрия), 1 — «дойдёт минимум раз» (команды и события механики; получатель обязан быть идемпотентным — переживать дубль), 2 — дорого, не используем.
- **Retain** — брокер хранит последнее сообщение топика: новый подписчик сразу узнаёт текущее состояние (статусы узлов).
- **LWT (last will)** — «завещание»: брокер сам публикует «узел offline», если ESP32 пропал. Так мы бесплатно получаем мониторинг живости механики.
- Формат payload, схема топиков и версии — контракт в `docs/домен-модель-и-контракты.md` + `hardware/integration/`. Менять втихую нельзя.

## 4.4 Камеры: полоса и декодирование (почему это вообще вопрос)

- Один поток 1080p H.264 ≈ 2–4 Мбит/с. 80 камер ≈ 160–320 Мбит/с — гигабитная LAN справляется, но **декодировать** 80 потоков одному миникомпу тяжело.
- Наш паттерн: CV не нужны 30 fps — нужен **кадр раз в 10–30 с** с принтера (и чаще при подозрении). Поэтому edge не держит все потоки постоянно, а забирает кадры по расписанию; постоянная запись — только видеобуфер вокруг инцидентов.
- USB против IP: см. блок 3.6. Стенд — USB; ферма — ❓ решение аудита (замер сети площадки + расчёт декодирования).

## 4.5 WireGuard и безопасность

- Туннель поднимает **площадка → центр** (исходящее соединение): на роутере клиента не открывается ни один входящий порт. Промзонный NAT/провайдер — не проблема.
- Через туннель ходят: синхронизация журнала, кадры для тяжёлой CV/VLM, панель→edge команды, наш удалённый доступ (роль «инженер поддержки»).
- Обрыв туннеля ≠ остановка фермы: правила деградации (ТЗ-1 разд. 4) — edge доигрывает очередь сам.
- Секреты (ключи WireGuard, токены MQTT) — не в коде и не в git; аудит-лог всех команд с указанием инициатора.

## 4.6 Чек-лист сетевого аудита площадки (веха 0)

1. Схема помещения: где стойки, где розетки Ethernet, длина трасс (решает USB vs IP камеры).
2. Роутер/провайдер: полоса, NAT, стабильность (пинг-лог на сутки), возможность второго канала (LTE-резерв — опция).
3. Сколько принтеров достижимо проводом, сколько только Wi-Fi; качество эфира.
4. Электрика: фазы, автоматы, заземление, где ИБП.
5. Место edge-узла: питание, провод до коммутатора, физическая защита (не «на столе у оператора»).
