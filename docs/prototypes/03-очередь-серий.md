# Прототип 03. Очередь серий

Родительский экран: `docs/UX-видение-панели.md`, раздел 5.7.

Цель: сделать диспетчер предсказуемым. Оператор должен понимать, что печатается сейчас, что пойдет дальше, почему часть заданий ждет и хватает ли очереди на ночной режим.

## Состояние для тестового макета

Выбрано состояние: **очередь покрывает ночь с запасом, но одна серия блокируется материалом**.

Данные:
- покрытие ночи: 18 ч 40 мин / 12 ч;
- в работе: 4 задания;
- следующее: 3 задания;
- заблокировано: 1 задание;
- edge mirror: актуален, last sync 14:26.

## Поверхности и блоки

```text
PAGE SURFACE
  TOP STATUS SURFACE
  LEFT NAV
  MAIN
    HERO: "Очередь серий" + покрытие ночи
    QUEUE GROUP: выполняется
    QUEUE GROUP: следующие задания
    QUEUE GROUP: заблокировано
    PREDICTABILITY PANEL: почему диспетчер выбрал порядок
  RIGHT PANEL
    выбранное задание
    совместимость
    блокеры
    действия
```

## Ключевые действия

- reorder queue;
- создать серию;
- изменить количество;
- отменить задание;
- исключить из ночи;
- retry print;
- открыть изделие/принтер.

## Backend handoff

Сущности:
- Job;
- Product;
- Printer;
- Spool;
- SliceArtifact;
- OperatorTask.

Команды/события:
- reorder queue;
- create job;

- retry print;
- assign job;
