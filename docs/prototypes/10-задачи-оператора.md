# Прототип 10. Задачи оператора

Родительский экран: `docs/UX-видение-панели.md`, раздел 5.9.

## Сценарий макета

Оператор открывает один список вместо обхода фермы. Приоритет строится по тому, что разблокирует поток и ночной протокол; выбрана задача освобождения тары на стойке A.

## Композиция

- Фильтры отделяют задачи ночи, инциденты и регламент.
- Каждая строка: физический объект → действие → причина → результат разблокировки.
- Rail раскрывает непосредственный чек-лист и ожидаемое подтверждение hardware.

## Проверяемые решения

- Нельзя закрыть физическую задачу простым кликом, если нужно показание датчика.
- Просрочка и связь с ночным блокером не теряются в общем списке.
- Оператор видит, что изменится после выполнения, до того как возьмет задачу.
