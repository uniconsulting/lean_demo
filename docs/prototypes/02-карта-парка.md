# Прототип 02. Карта парка

Родительский экран: `docs/UX-видение-панели.md`, раздел 5.2.

Цель: показать оператору физическую картину фермы и отклонения от нормы без чтения логов. Экран должен отвечать:
- какие стойки и принтеры сейчас в работе;
- где требуется человек;
- какие устройства автономные, а какие только `Контроль`;
- какой физический объект связан с блокером;
- куда перейти для действия.

## Состояние для тестового макета

Выбрано состояние: **дневной режим после ночи, один блокер на стойке A и 2 control-only принтера**.

Данные:
- стойка A: 4 автономных K1C, K1C-04 ждет освобождения тары на шаге `confirm_drop`;
- стойка B: 4 автономных K1C, все принтеры прошли preflight;
- зона Control-only: 2 Bambu под камерой/реле/учетом без команд прошивки;
- фильтр по умолчанию: `все`, но виден быстрый фильтр `только требуют человека`.

## Поверхности и блоки

```text
PAGE SURFACE
  TOP STATUS SURFACE
  LEFT NAV
  MAIN
    HERO: "Карта парка" + агрегаты по статусам
    FILTER STRIP: стойка / режим / материал / статус / только блокеры
    RACK A SURFACE
      printer cards: camera + FSM + job + material + blocker
    RACK B SURFACE
      printer cards
    CONTROL-ONLY SURFACE
      camera cards + ручная отметка занятости
  RIGHT PANEL
    выбранный принтер K1C-04
    next action
    причины блокера
    события
```

Радиусы: каждый `printer card` наследует radius от поверхности своей стойки: `R2 = R1 - I`.

## Ключевые действия

- открыть карточку принтера;
- отфильтровать только блокеры;
- открыть live-кадр;
- создать задачу обслуживания;
- вручную отметить занятость control-only принтера;
- перейти к инциденту или очереди.

## Backend handoff

Сущности:
- Printer;
- Rack;
- Site;
- Spool;
- Job;
- Incident;
- OperatorTask.

Команды/события:
- update printer state;
- open printer details;
- create operator task;
- manual busy mark;
- relay power request;
- sync camera status.

Минимальные поля:
- printer mode: autonomous / control_only;
- FSM state;
- physical location: site/rack/slot;
- current job;
- loaded spool;
- camera status;
- next_action;
- blocker reason.
