# Прототип 04. Зарядить ночь

Родительский экран: `docs/UX-видение-панели.md`, раздел 5.10.

Цель: главный MVP-сценарий панели. Экран должен честно ответить, можно ли оставить стойку автономии на 12 часов без человека, и если нельзя — показать конкретные блокеры и действия.

## Состояние для тестового макета

Выбрано состояние: **ночь не готова, 2 физически подтверждаемых блокера**.

Данные:
- покрытие очереди: 18 ч 40 мин / 12 ч;
- материал: катушка #17 не покрывает серию #024;
- тара годных на стойке A заполнена;
- preflight прошли 7/8 автономных принтеров; K1C-04 ждет тару;
- edge и пожарная цепь в норме;
- center online, деградация не активна.

## Поверхности и блоки

```text
PAGE SURFACE
  TOP STATUS SURFACE
  LEFT NAV
  MAIN
    HERO: "Ночь не готова" + coverage
    CHECKLIST SURFACE
      очередь / филамент / тара / preflight / fire / edge / center
    BLOCKERS SURFACE
      blocker cards with object -> reason -> action
    NIGHT POLICY SURFACE
      что будит / что не будит / что пропускается
  RIGHT PANEL
    primary next action
    approve night disabled
    exclude printer option
```

## Ключевые действия

- approve night;
- replace spool;
- mark bin emptied;
- run preflight;
- exclude printer from night;
- add jobs;
- open blocker.

## Backend handoff

Сущности:
- Job;
- Printer;
- Spool;
- BuildSurface/GlueUnit;
- OperatorTask;
- ShiftReport;
- Incident.

Команды/события:
- approve night;
- exclude printer;
- run preflight;
- mark bin emptied;
- replace spool;
- queue coverage calculated;
- degradation policy applied.
