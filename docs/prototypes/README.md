# UI/UX low-fi прототипы ЛЁН

Пакет статических HTML-макетов для ручного ревью операционных сценариев MVP. Это не финальный visual design и не контракт API: он проверяет порядок действий, объяснимость состояния и связность производственного потока.

Выбранный референс геометрии и компоновки: `references/shift-summary-selected-reference.png`. Его темно-синий базовый тон намеренно не переносится: темы построены на нейтральном графите и бело-серых поверхностях.

## Быстрый запуск

Открыть `index.html` в браузере: входная точка сразу перенаправляет в `01-shift-summary.html`. Отдельной страницы-карты прототипов больше нет; переходы между маршрутами доступны из общей левой навигации.

## Принцип построения

Главный язык панели — производственный поток:

```text
изделие → слайсинг → серия → очередь → печать → контроль → снятие → сортировка → журнал → отчет
```

На каждом экране сохраняется одна и та же логика: **статус → причина/источник → следующее действие → аудит/журнал**. Экран не должен превращаться в мониторинг ради мониторинга.

## Surface system и темы

- Ночная тема использует нейтральный графитовый canvas и темно-серые поверхности без темно-синего подтона.
- Светлая тема использует белый canvas, белые фреймы и холодные светло-серые вложенные поверхности.
- Цвет применяется семантически: зеленый — норма, янтарный — ожидание/задача, красный — ошибка, приглушенный голубой — информационный переход.
- Для фреймов, карточек и модулей не используются обводки. Техническое разделение допустимо внутри таблиц и логов, если без него теряется читаемость.
- Базовый ритм: `4px` primitive и `8px` major scale.
- Размеры shell, spacing и radius адаптивны через `clamp()`.
- Базовый внешний radius после выбора плотного операторского макета — `18–24px`.
- Вложенный radius считается от ближайшего смыслового фрейма: `R2 = R1 - I`, где `I` — inset между ними. Поэтому визуально вложенный блок продолжает дугу своего родителя, а не общего canvas.
- Переключатель темы доступен в верхней командной строке; выбор сохраняется для всех связанных экранов.
- Тема применяется через `prototype-init.js` до загрузки основного CSS, поэтому при переходах нет вспышки другой палитры.
- Внутренние переходы используют короткий fade-out и CSS View Transitions там, где браузер их поддерживает; `prefers-reduced-motion` отключает анимацию.

## Иконки

- Используется локально сохраненная библиотека **Lucide 1.24.0**: `vendor/lucide/lucide.min.js`.
- Лицензия библиотеки хранится рядом: `vendor/lucide/LICENSE`.
- Навигация использует отдельные метафоры для сводки, фермы, принтеров, очереди, изделий, качества, обслуживания, событий, аналитики и настроек.
- Та же библиотека заменяет заглушки темы, файла, камеры, чек-листов, предупреждений, действий и внешних переходов.

## Что проверять на ручном ревью

1. Пройти девять маршрутов из раздела 9 `docs/UX-видение-панели.md`.
2. Для каждого экрана назвать следующее действие, его физический объект и причину — без обращения к документации.
3. Проверить, что деструктивные действия обещают подтверждение и аудит, а политика ABS/ASA не допускает автоматическую паузу.
4. Проверить состояния деградации: edge продолжает работу, журнал и очередь синхронизируются позже.
5. Отдельно оценить визуальную систему: нет ли лишних фреймов, читается ли плотность данных, логичны ли скругления вложенных поверхностей.

До ручного ревью Д + Ф весь пакет имеет статус `нужен тест`.
